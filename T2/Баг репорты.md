## Bug Report: Поиск не находит файлы, находящиеся в корзине

**ID:** `UI-SEARCH-001`  
**Название бага:** Файлы, находящиеся в корзине, не отображаются в поиске по всему диску  
**Серьезность:** Major
**Приоритет:** High
**Автор:** Жуклин Никита
**Окружение:**

- Браузер: Google Chrome 138.0.7204.97
- ОС: Windows 11 x64
- Аккаунт Яндекс.Диск: обычный (не бизнес)
---

### Шаги для воспроизведения:

1. Загрузить файл на Яндекс.Диск (например, test.txt)
2. Удалить файл, переместив его в корзину
3. Вернуться на главную страницу Яндекс.Диска
4. Ввести имя удалённого файла в поле поиска (например, test)
5. Нажать Enter

### Ожидаемый результат:

Файл из корзины отображается в результатах поиска

### Фактический результат:

Файл не отображается в результатах поиска. Он виден только при ручном переходе в раздел "Корзина"

### Дополнительная информация:

- В текущей реализации файл можно найти только при выполнении поиска внутри раздела "Корзина". При поиске по всему диску такие файлы не отображаются. Это может свидетельствовать о том, что поиск по корзине работает как отдельный механизм и не включается в глобальный поиск по умолчанию.
- В таком случае поведение системы расходится с требованиями, и необходимо:
  - либо изменить реализацию глобального поиска, чтобы включать корзину,
  - либо уточнить требования, что поиск по корзине возможен только при нахождении в этом разделе.

## Bug Report: Возвращается неверный код ответа при успешном создании папки

**ID:** `API-FOLDER-001`  
**Название бага:** При успешном создании папки API возвращает код 201 вместо ожидаемого 200  
**Серьезность:** Minor  
**Приоритет:** Medium  
**Автор:** Жуклин Никита
**Окружение:**

- Операционная система: Windows 11 x64
- REST-клиент: Postman 11.52.4
- API Яндекс.Диск: `PUT https://cloud-api.yandex.net:443/v1/disk/resources`
- Авторизация: OAuth-токен пользователя
- Тестовый аккаунт Яндекс.Диска: обычный (не бизнес)
---

### Предусловия:

- Пользователь авторизован и имеет действующий OAuth-токен
- На Диске нет папки по пути `/test-folder`

### Шаги для воспроизведения:

1. Отправить `PUT`-запрос на эндпоинт
   `https://cloud-api.yandex.net/v1/disk/resources?path=/test-folder`
2. Убедиться, что заголовки содержат:
  - `Authorization: OAuth <валидный_токен>`
  - `Content-Type: application/json`
3. Отправить запрос

### Ожидаемый результат:

- Код ответа: `200 OK`
- Папка создаётся по указанному пути
- Ответ содержит ссылку на созданный объект (папку), которую можно использовать для получения полной информации.

### Фактический результат:

- Код ответа: `201 Created`
- Папка действительно создаётся
- Ответ корректен по содержанию

### Дополнительная информация:

- В документации к API указано, что при успешном создании папки должен возвращаться код `200 OK`
- Возврат `201 Created` соответствует документации, но противоречит явно указанным требованиям
- Необходимо:
  - Либо скорректировать требования и указать, что возвращается `201 Created`,
  - Либо изменить реализацию сервиса и возвращать `200 OK`, как заявлено.


## Bug Report: Возвращается неверный код ответа при попытке создать уже существующую папку

**ID:** `API-FOLDER-002`  
**Название бага:** При создании папки по уже существующему пути API возвращает код 409 вместо ожидаемого 400  
**Серьезность:** Minor  
**Приоритет:** Medium  
**Автор:** Жуклин Никита
**Окружение:**

- Операционная система: Windows 11 x64
- REST-клиент: Postman 11.52.4
- API Яндекс.Диск: `PUT https://cloud-api.yandex.net:443/v1/disk/resources`
- Авторизация: OAuth-токен пользователя
- Тестовый аккаунт Яндекс.Диска: обычный (не бизнес)
---

### Предусловия:

- Пользователь авторизован и имеет действующий OAuth-токен
- На Диске уже существует папка по пути `/test-folder`

### Шаги для воспроизведения:

1. Отправить `PUT`-запрос на эндпоинт  
   `https://cloud-api.yandex.net/v1/disk/resources?path=/test-folder`
2. Убедиться, что заголовки содержат:
  - `Authorization: OAuth <валидный_токен>`
  - `Content-Type: application/json`
3. Отправить запрос

### Ожидаемый результат:

- Код ответа: `400 Bad Request`
- В теле ответа — информация об ошибке создания, структура:
  `{   "message": "string",   "description": "string",   "error": "string" }`

### Фактический результат:

- Код ответа: `409 Conflict`
- Тело ответа:
  `{   "error": "DiskPathPointsToExistentDirectoryError",   "description": "Specified path \"/test-folder\" points to existent directory.",   "message": "По указанному пути \"/test-folder\" уже существует папка с таким именем." }`

### Дополнительная информация:

- Согласно текущим требованиям, при повторной попытке создать папку по уже существующему пути должен возвращаться код `400 Bad Request`. Однако API возвращает код `409 Conflict`, что корректно с точки зрения документации, но противоречит заявленным требованиям
- Необходимо:
  - Либо уточнить требования и заменить 400 на 409, если это поведение считается корректным
  - Либо изменить реализацию API, чтобы возвращать `400 Bad Request`, как указано в требованиях
